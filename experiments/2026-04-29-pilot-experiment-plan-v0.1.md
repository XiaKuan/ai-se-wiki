**研究问题：** 不同的时序 metrics 可视化渲染策略如何影响 VLM 在 RCA 任务中的准确率？

---

## 一、实验概述

### 1.1 背景与目标

**研究空白：** CCF-A 论文中，没有人系统研究过"metrics 可视化渲染策略对 RCA 准确率的影响"。

**预实验目标：**
1. 验证"VLM 能通过可视化图像完成 RCA"这一基础假设
2. 初步探索渲染策略各维度的效应量（方向性）
3. 识别实验流程瓶颈（图像生成、API 调用、数据标注）

### 1.2 核心假设（H1-H5）

| ID | 假设内容 | 可证伪条件 |
|----|----------|------------|
| H1 | VLM 对折线图的 RCA 准确率显著高于热力图和箱线图 | 折线图与热力图差异 < 5% |
| H2 | 存在最优时间窗口粒度，且因异常类型而异 | 不同窗口差异 < 3% |
| H3 | 多指标叠加图的 RCA 准确率低于单指标分别展示 | 叠加图准确率 ≥ 单指标平均 |
| H4 | 异常高亮（红色）能显著提升 VLM 聚焦能力 | 高亮 vs 灰度差异 < 3% |
| H5 | 适量统计标注能提升准确率，过度标注会下降 | 适量 vs 无标注差异 < 2% |

### 1.3 实验设计

**设计类型：** 单一变量对照实验（固定 VLM，改变渲染策略）

**固定变量：** GPT-4V（统一版本）

**自变量：** 渲染策略（5维度）

**因变量：** Top-1 Accuracy、Recall@3、MRR

| 指标 | 定义 | 预实验阈值 |

## 十一、未决事项（Open Questions）

### 11.1 实验设计类

- [ ] **OQ1**: 是否需要在 Prompt 中加入"图表类型"的显式说明？
  - 例：告知 VLM "这是一张折线图"
  - 可能影响：引入额外信息干扰

- [ ] **OQ2**: 多指标叠加时，指标数量是否需要固定？
  - 当前设计：双指标叠加
  - 备选：固定选择"最可能异常的2个指标"

- [ ] **OQ3**: 异常区域的标注方式是否需要统一？
  - 当前设计：红色高亮
  - 问题：不同图表类型的异常表示方式不同

### 11.2 评估方法类

- [ ] **OQ4**: RCAEval 的 ground truth 是否需要二次标注？
  - 问题：RCAEval 的标注可能存在歧义
  - 影响：评估结果可信度

- [ ] **OQ5**: 如何处理 VLM 输出格式不匹配的情况？
  - 例如：服务名称与 RCAEval 标注不完全一致
  - 需要：模糊匹配 or 严格匹配

### 11.3 资源类

- [ ] **OQ6**: GPT-4V API 额度是否充足（~$30）？
- [ ] **OQ7**: 图像存储空间是否足够（~900 张 × ~100KB = ~90MB）？
